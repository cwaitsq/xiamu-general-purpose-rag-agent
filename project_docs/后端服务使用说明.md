# 后端服务使用说明

## 1. 这层是干什么的

后端服务是业务入口，主要做 4 件事：

1. 接前端、Postman、n8n 的请求
2. 调 Gateway 做 RAG 问答
3. 接收知识资料上传，触发入库
4. 记录会话、知识任务、问答日志

当前默认地址：

```text
http://127.0.0.1:8877
```

## 2. 默认配置

- 默认租户：`foreign_trade_demo`
- 默认数据库：`SQLite`
- 默认上传目录：`tenant_kb/<tenant_id>/raw`
- 默认入库状态：`active`
- 默认刷新索引：`true`

如果要切 PostgreSQL，可以配：

- `BACKEND_DB_DRIVER=postgres`
- `BACKEND_DATABASE_URL=postgresql://...`

注意：

- 切 PostgreSQL 前要先执行：

```powershell
python -m pip install -r backend_service\requirements.txt
```

- 当前本地环境默认还是 `sqlite`

## 3. 启动命令

先装后端依赖：

```powershell
python -m pip install -r backend_service\requirements.txt
```

```powershell
python tools\启动_backend.py
```

健康检查：

```text
GET http://127.0.0.1:8877/health
```

## 4. 主要接口

问答接口：

```text
POST /api/chat/send
```

请求示例：

```json
{
  "tenant_id": "foreign_trade_demo",
  "session_id": "sess_001",
  "question": "最小起订量是多少？",
  "top_k": 5
}
```

会话历史：

```text
GET /api/chat/history?tenant_id=foreign_trade_demo&session_id=sess_001
```

上传知识资料：

```text
POST /api/knowledge/upload
```

支持两种方式：

- `multipart/form-data` 直接传文件
- `application/json` 直接传本地文件路径

JSON 示例：

```json
{
  "tenant_id": "foreign_trade_demo",
  "file_path": "C:/Users/zgy/Desktop/通用对话/tenant_kb/foreign_trade_demo/raw/外贸客服样例/客户常见问题_原稿.txt",
  "file_name": "客户常见问题_原稿.txt"
}
```

创建入库任务：

```text
POST /api/ingestion/jobs
```

查询任务：

```text
GET /api/ingestion/jobs/{job_id}?tenant_id=foreign_trade_demo
```

查询知识列表：

```text
GET /api/knowledge/list?tenant_id=foreign_trade_demo
```

查询问答日志：

```text
GET /api/logs/qa?tenant_id=foreign_trade_demo&limit=100
```

## 5. 数据现在存哪里

数据库里存这些：

- 会话
- 消息历史
- 知识文档清单
- 入库任务
- 问答日志
- 转人工日志

原始资料和整理结果存这些目录：

```text
tenant_kb/<tenant_id>/raw
tenant_kb/<tenant_id>/docs
tenant_kb/<tenant_id>/docs/auto
tenant_kb/<tenant_id>/prepare_output
tenant_kb/<tenant_id>/ingest_output
```

## 6. 这层怎么跟 Gateway 配合

问答链路：

```text
前端 / n8n / Postman
-> /api/chat/send
-> Gateway /query
-> qmd 检索
-> 大模型生成
-> 后端落会话和 qa_logs
```

入库链路：

```text
前端 / Postman 上传资料
-> /api/knowledge/upload
-> 保存到 tenant_kb/<tenant>/raw
-> 调 Gateway /ingest/rebuild
-> 整理、切片、刷新 qmd
-> 回写 knowledge_docs 和 ingestion_jobs
```

## 7. 安全控制

- 按租户隔离会话和知识
- 上传文件按租户分目录
- 问答日志做密钥脱敏
- 请求体大小有限制
- 上传文件大小有限制
- 基础限流已打开
