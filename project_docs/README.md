# 项目文档导航

`project_docs/` 用于集中存放本仓库的正式项目文档。

这里的文档主要面向以下几类读者：

- 需要快速理解项目结构与能力边界的开发人员
- 需要部署、联调和交付系统的实施人员
- 需要查看能力说明、迁移说明和维护约定的项目负责人

本目录不用于存放零散草稿、临时会议记录或一次性测试说明。

## 1. 建议阅读顺序

如果是第一次接手这个项目，建议按下面顺序阅读：

1. 根目录 `README.md`
2. `仓库迁移与升级说明.md`
3. `任务型Agent能力说明.md`
4. `部署文档_v1.md`
5. `开发环境搭建文档_v1.md`
6. `后端服务使用说明.md`

## 2. 文档职责说明

| 文档 | 作用 |
|---|---|
| `仓库迁移与升级说明.md` | 说明仓库演进背景、升级方向与主仓维护策略 |
| `任务型Agent实施提示词.md` | 记录任务型 Agent 设计与实施阶段可复用的提示词材料 |
| `任务型Agent能力说明.md` | 说明任务型 Agent 当前能力范围、闭环方式与扩展方向 |
| `后端服务使用说明.md` | 说明后端接口、运行方式与联调要点 |
| `开发环境搭建文档_v1.md` | 说明本地开发环境如何搭建 |
| `部署文档_v1.md` | 说明部署过程、依赖与上线注意事项 |
| `项目收口说明.md` | 说明项目阶段性收口内容与当前交付边界 |

## 3. 使用建议

- 如果你要理解为什么这个仓库会拆成前端、后端、Gateway、知识库和工作流几层，优先看根目录 `README.md`。
- 如果你要部署系统，优先看 `部署文档_v1.md` 与 `开发环境搭建文档_v1.md`。
- 如果你要扩展任务型 Agent，优先看 `任务型Agent能力说明.md`。
- 如果你要解释仓库来历、迁移关系和后续主维护方向，优先看 `仓库迁移与升级说明.md`。

## 4. 维护原则

- 文档默认使用中文维护。
- 目录内只保留长期有效、需要随仓库一起交付或维护的正式资料。
- 重大架构变更、部署方式调整、模块边界变化，应同步更新本目录文档。
