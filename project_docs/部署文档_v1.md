# 部署文档 v1

## 1. 部署目标

把这套外贸客服 RAG 系统部署成可长期运行的环境，包含：

- 前端
- 后端
- Gateway Runner
- n8n
- qmd 本地索引
- 租户知识目录

## 2. 当前推荐部署形态

第一版推荐这样分层：

1. `n8n`
2. `Gateway Runner`
3. `Python 后端`
4. `前端`

当前默认不把 `Qdrant` 放进主链路。
如果后面要做第二版检索增强，再把它作为扩展能力接入。

## 3. 端口规划

- 前端：`3001`
- 后端：`8877`
- Gateway Runner：`8765`
- n8n：`5678`
- PostgreSQL：`5432`

## 4. 环境变量

至少要配这些：

- `RAG_KEFU_GATEWAY_API_KEY`
- `QMD_COLLECTION`
- `LLM_API_KEY`
- `LLM_MODEL`
- `DASHSCOPE_API_KEY`
- `DEFAULT_TENANT_ID`
- `BACKEND_GATEWAY_BASE_URL`

如果后端切 PostgreSQL，再配：

- `BACKEND_DB_DRIVER=postgres`
- `BACKEND_DATABASE_URL=postgresql://...`

## 5. 目录说明

正式环境至少要保留这些目录：

```text
tenant_kb/
  <tenant_id>/
    raw/
    docs/
    docs/auto/
    prepare_output/
    ingest_output/

.qmd/
frontend/
backend_service/
gateway/
```

## 6. 启动顺序

```text
1. PostgreSQL（如果启用）
2. n8n
3. Gateway Runner
4. 后端
5. 前端
```

## 7. 依赖安装

先装 Python 依赖：

```powershell
python -m pip install -r requirements.txt
```

## 8. 启动命令

Gateway Runner：

```powershell
python C:\Users\zgy\Desktop\factory_n8n\tools\gateway-runner.py --repo-root C:\Users\zgy\Desktop\通用对话 serve --host 0.0.0.0 --port 8765
```

后端：

```powershell
python tools\启动_backend.py
```

前端：

```powershell
cd frontend
pnpm start --port 3001
```

## 9. 停止方式

先停业务层：

1. 前端
2. 后端
3. Gateway Runner

再停基础依赖：

4. n8n
5. PostgreSQL

## 11. 上线前检查

- `.env` 已补齐
- Gateway 能回答问题
- 后端能写会话和日志
- 上传资料后能完成整理、切片、qmd 刷新
- 租户目录隔离正常
- 大模型 Key 和多模态 Key 已配置
- 日志里没有明文密钥

## 12. 风险提示

- PostgreSQL 驱动已经放进 `backend_service/requirements.txt`，切库前要先执行依赖安装
- 资料更新后必须刷新 qmd，不然新知识不会立刻生效
- 内部知识虽然会入库，但外部问答默认不应返回
