# 开发环境搭建文档 v1

## 1. 目标

这份文档只讲一件事：把本项目在本地跑起来。

## 2. 必备环境

- Windows 10/11
- Node.js `20.x`
- Python `3.10+`
- Docker Desktop
- n8n
- qmd 运行依赖

可选环境：

- PostgreSQL `14+`

## 3. 当前本地端口

- 前端：`3001`
- n8n：`5678`
- Gateway Runner：`8765`
- 后端：`8877`

## 4. 配置文件

项目里现在有两份配置来源：

1. 根目录 `.env`
2. `gateway/.env`

建议：

- 平时只维护根目录 `.env`
- `gateway/.env` 只做 Gateway 单独覆盖

## 5. Python 依赖

建议先建虚拟环境：

```powershell
python -m venv .venv
.venv\Scripts\activate
```

现在已经补了统一依赖文件，直接装就行：

```powershell
python -m pip install -r requirements.txt
```

如果你只想单独装某一层，也可以：

```powershell
python -m pip install -r gateway\requirements.txt
python -m pip install -r backend_service\requirements.txt
```

## 6. Node 依赖

前端安装依赖：

```powershell
cd frontend
pnpm install
```

如果没有 `pnpm`：

```powershell
npm install -g pnpm
```

## 7. qmd 相关

当前方案默认用本地 qmd 做检索。

需要保证：

- `node` 能用
- `qmd_repo/dist/cli/qmd.js` 存在

Gateway 会自动优先读取：

- 根目录 `.env` 里的 `QMD_CLI_PATH`
- 如果没配，再尝试仓库里的默认 qmd 路径

## 8. n8n

当前本地用浏览器访问：

```text
http://127.0.0.1:5678
```

如果要本地单独启动：

```powershell
n8n
```

## 9. PostgreSQL

当前不是必需。

如果你要按企业版方式联调数据库，可以用 Docker 起一个：

```powershell
docker run -d --name rag-postgres -e POSTGRES_PASSWORD=postgres -e POSTGRES_DB=rag_app -p 5432:5432 postgres:14
```

然后在 `.env` 里补：

```text
BACKEND_DB_DRIVER=postgres
BACKEND_DATABASE_URL=postgresql://postgres:postgres@127.0.0.1:5432/rag_app
```

## 10. 启动顺序

推荐顺序：

1. n8n
2. Gateway Runner
3. 后端
4. 前端

## 11. 启动命令

启动 Gateway Runner：

```powershell
python C:\Users\zgy\Desktop\factory_n8n\tools\gateway-runner.py --repo-root C:\Users\zgy\Desktop\通用对话 serve --host 0.0.0.0 --port 8765
```

启动后端：

```powershell
python tools\启动_backend.py
```

启动前端：

```powershell
cd frontend
pnpm dev --port 3001
```
